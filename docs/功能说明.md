# Dubai Property English Coach 功能说明

## 目标

给迪拜房产业务运营人员做一个碎片化英语学习工具。学习内容来自真实工作场景，不做普通背单词，重点帮助用户在句子、对话和任务里掌握英语。

## 知识库分类

知识库内部按业务类型组织，但学员学习时不手动切换分类，而是进入学习后按“上一个 / 下一个”连续学习。分类只作为当前知识点标签展示。

1. 线索沟通
   - TikTok 评论线索打招呼
   - 判断 buy / rent
   - 询问预算、区域、户型
   - 跟进未回复客户

2. Agent 协作
   - 约会议
   - 催房源视频和楼盘资料
   - 同步线索进展
   - 沟通看房、客户反馈、下一步安排

3. 新伙伴合作
   - 介绍我们的合作方式
   - 说明我们做线索收集和清洗，不承诺房源成交
   - 沟通分佣、独家线索、合作流程

4. 楼盘介绍
   - 介绍位置、户型、价格、付款计划
   - 说明投资和自住场景
   - 回答客户常见问题

5. 内容运营
   - 和 agent 沟通拍摄脚本、房源视频、户型图、价格和付款计划
   - 生成视频 caption 和发布前素材检查

6. 团队日常沟通
   - 同步今日进展、卡点、优先级
   - 跟进任务、确认下一步

7. WhatsApp / 客户跟进
   - 老线索轻量跟进
   - 客户忙、预算低、不信任、比较项目等常见回复

## 学习流程

第一版 MVP 有两个主模块，以及一个常驻 AI 老师：

1. 学习
   - 给出真实对话
   - 拆解核心句子
   - 提炼关键词
   - 增加常用句库
   - 读音不做独立模块：英文单词可点击朗读，每句话后有喇叭图标可读整句
   - 标出容易出错的表达
   - 页面按“案例对话、为什么这么说、关键词/句库、易错点、可直接用”分区，降低信息混乱
   - 不展示分类 tab；学员只需要顺序学习，可无限上一个 / 下一个
   - 学会后点击“标记学会了”，该知识点加入应用考试池
   - 学习库来源包含通用知识点和个人知识点
   - AI 老师生成的案例会加入个人学习库，后续也能被学习和检测

2. 检测
   - AI 扮演客户、agent 或合作伙伴
   - 基于用户已标记学会的知识点出题
   - 以 4 轮连续对话考试进行，不是一问一答
   - 用户每轮用英文回复
   - AI 会根据用户上一轮回复继续追问
   - 系统指出问题，并改写为更自然的业务英语
   - 重点检查是否表达清楚、是否礼貌、是否符合业务边界
   - 考试模式隐藏示范答案、关键词和常用句库，避免变成开卷考试
   - 考试结果会保存到学员档案，后续可用于复盘
   - 界面以模拟对话为主体，知识点只作为旁边 tips

3. 常驻 AI 老师
   - 用户粘贴日常工作中看到的英文句子、单词或自己的问题
   - 也可以直接输入中文，让 AI 生成自然英文表达
   - 不再区分“中文转英文 / 英文转中文”按钮，由老师自动判断该怎么处理
   - AI 先判断需求，再翻译、解释、润色或生成话术
   - 解释为什么这样说，哪些词适合当前业务场景
   - 自动生成一个贴近业务的学习案例
   - 解析结果可加入“我的学习库”，成为个人知识点
   - 同一个案例不能重复加入

4. 管理员用户管理
   - 管理员账号用于开通学员账号
   - 管理员只需要设置用户名和密码
   - 每个学员登录后拥有独立学习档案
   - 学习进度、考试记录、保存案例跟随账号保存，不只保存在单台电脑浏览器
   - 用户管理不和学习/检测并列，管理员从右上角独立入口进入

## Token 消耗边界

不消耗 AI token：
- 登录、退出、管理员创建用户
- 学习页展示知识点、上一个 / 下一个
- 标记学会、保存学习进度
- 单词点读、句子朗读
- 查看学习库、导出 / 导入备份

会消耗 AI token：
- 检测里的动态多轮追问
- 检测结束后的对话分析和表达改写
- AI 老师问答、解析、中译英、英译中、生成个人学习知识点

后续资料库扩展：
- 学习资料库可以接入 Musewen 线上库，优先作为知识库来源同步到本地/数据库
- 同步后的学习资料仍按静态知识点展示，不在学员浏览学习页时临时调用 AI
- 冷启动阶段可以继续由 Codex 人工/半自动整理知识点写入静态库，不消耗网站 gptsapi 额度
- 后台知识库工厂上线后，再支持抓取、去重、AI 批量生成、审核发布

## 当前网页端可用状态

- 可以作为账号制网页工具使用：登录、学习、检测、AI 老师、学习档案、单词点读和句子朗读都可在浏览器里完成。
- 学习页当前采用轻量学习工作台界面：左侧是学习库和进度，中间是当前知识点和对话学习，右侧是当前知识点与最近检测。
- 学习页主线收敛为 4 个学习区：场景对话、表达拆解、关键词/常用句、替换练习。
- 学习页对话区采用左右气泡式剧本阅读：客户内容靠左、推荐回复靠右，避免像后台表格一样铺满整行；备份入口收进侧边栏的“数据备份”折叠项，减少对学习主流程的干扰。
- 当前视觉方案已接入 Claude 重新设计的 CSS：页面整体收窄、顶部标题降低压迫感、学习内容以中间卡片为主、右侧保留当前知识点和最近检测信息卡，AI 老师抽屉保持常驻入口。
- 读音交互：主推荐回复使用“听完整回复”按钮；关键词保留整词点读；长句不再拆成单词按钮，避免英文被挤成字母散落。
- 当前 GitHub Pages 版本不运行后端函数，所以 AI 老师会走本地兜底案例；正式 AI 后端需要部署到支持函数的平台，如 Cloudflare Pages、Vercel 或恢复 Netlify。
- 学员记录已支持账号档案，正式线上以 Cloudflare D1 为准；本地浏览器仍保留备份能力。
- 如果不想把 gptsapi key 写入 Cloudflare，可以使用本地 AI 代理：运行 `npm run local` 后打开 `http://127.0.0.1:8788`，网页会调用本机 `/api/ai-teacher`，key 只留在本机 `.env.local` 或本地环境变量里。
- 当前本地 AI 代理已验证可用：`gptsapi / gpt-4.1-mini` 能返回真实学习案例，`fallback=false`。
- 正式多人使用方案：Cloudflare Pages + D1 登录账号。只有登录用户才能进入工具和调用 `/api/ai-teacher`，管理员可以给别人开账号。

## 模型选择规则

- 低风险、批量、学习类任务：优先使用 opencode 免费模型
  - 默认模型：`deepseek-v4-flash-free`
  - 选择原因：这是当前线上测试中确认模型名存在的免费模型
  - 已测试不可用模型：`qwen3.6-plus-free`、`minimax-m3-free`，opencode 返回 model not supported
  - workspace：`https://opencode.ai/workspace/wrk_01KTT84SVS1P57GZ0HR62GT3YG`
  - 当前规则：opencode 收费模型暂不可用，只使用带 `-free` 的免费模型
  - 环境变量：`OPENCODE_API_KEY`
  - 可选覆盖：`OPENCODE_BASE_URL`、`OPENCODE_FREE_MODEL`
- gptsapi 付费模型不自动兜底
  - 默认模型：`gpt-4.1-mini`
  - 环境变量：`GPTSAPI_API_KEY`
  - 兼容旧变量名：`GPTAPI_API_KEY`
  - 可选覆盖：`GPTAPI_BASE_URL`、`GPTAPI_QUALITY_MODEL`
  - 前端“付费 AI”开关打开时，本次请求会显式传 `quality=paid`，后端直接调用 gptsapi
  - 前端开关关闭时，只调用 opencode 免费模型，不自动切到 gptsapi
- API key 只放 Cloudflare Secret / 平台环境变量，不写进前端代码和仓库。
- 当前 AI 老师、检测动态追问、检测反馈都走后端函数。
  - Netlify 路径：`/.netlify/functions/ai-teacher`
  - Vercel 路径：`/api/ai-teacher`
  - Cloudflare Pages 路径：`/api/ai-teacher`
  - 线上正式版优先部署到 Cloudflare Pages，因为它同时支持静态网页和后端函数。
- AI 老师支持连续对话：
  - `coach_chat`：根据前文继续回答用户追问
  - `coach_summarize`：把整段老师对话整理成一个个人学习案例，再加入学习库
  - 如果用户质疑老师表达是否自然或是否正确，老师需要先自查，再修正或给出更稳妥表达
  - 这两个动作会调用后端 AI；默认免费模式只走 opencode，打开“付费 AI”后才走 gptsapi
  - 普通学习页浏览和静态知识库不调用 AI，不消耗模型额度

## Token 消耗与模型路由

- 不消耗 token：
  - 顺序学习通用知识库
  - Musewen 静态知识库
  - 个人学习库浏览
  - 单词点读和句子朗读（浏览器本地语音）
- opencode 免费优先：
  - `coach_case`：单次 AI 老师案例生成
  - `coach_chat`：AI 老师连续对话
  - `coach_summarize`：把老师对话整理成学习案例
  - `exam_next`：检测里的下一轮模拟追问
  - `exam_feedback`：检测结束后的反馈
- gptsapi 付费：
  - 默认不自动调用
  - AI 老师抽屉提供“付费 AI”开关
  - 开关关闭：只走免费模型，失败时返回本地兜底
  - 开关打开：本次请求直接走 gptsapi，不先试免费模型，不做黑盒兜底
- 免费模型测试状态：
  - 2026-07-03：Cloudflare 线上函数已能请求 opencode
  - `qwen3.6-plus-free`：opencode 返回 model not supported
  - `minimax-m3-free`：opencode 返回 model not supported
  - `deepseek-v4-flash-free`：模型名存在，但当前返回 FreeUsageLimitError / 429，说明免费额度或频率受限
  - 因此免费模型通路已接通，但当前不能保证稳定可用；前端会显示 provider/model 和免费模型失败原因
- API key 管理：
  - 只放 Cloudflare Secret / 本地 `.env.local` / 平台环境变量
  - 不写入前端代码，不写入仓库，不写入文档明文
